Movielens
Redesigning Tagging Interface

Role: Researcher | UX Designer | Front End Developer

Team Size: 3      Date: 2013/8-2014/5

Category: UX DesignFront-End Development Website

Intro

Movielens is a personalized movie recommendation website created by Grouplens Research Lab. It used to collect data and conduct experiments for research on online community and recommendation system.

Tagging interface is a feature where users can rate each tag based on how it is related to the movie and to their own preference. Lots of the on-going research project conducted through the website was based on the data collected from tags.

Goal

Redesigning a tagging interface to fit the new interface but still include various features that supports research.

Challenge

Unlike traditional tag interface, Movielen's tag interface acquires lots of dynamics

  • Users should be able to apply affects on tags such as 'I like this tag about this movie' and 'I dislike this tag about this movie'

  • Users should be able to view community tags and the aggregate results of other people's affects for each tag.

So the final design needs to have a easy understandable interface to encourage users to use this feature. It also needs to be able to convey clearly what these different dynamics mean to better collect meaningful data.

My Role

As the UX designer, I define the interface and interaction, draw wireframes, conduct stakeholder interviews, and host brainstorming session to ideate a solution that fits the lab's need.

As the front-end developer, I worked with a backend developer to implement this interface and the interaction.

Design

Early Versions

In early versions, I tried to use more color and icon to convey the complex interactions.

Brainstorm

However, after brainstorming with researchers, I prioritized clarity as the key of design.

Wireframe

Then I redesigned the wireframe and use color to present aggregated affects.

Final Design

An interactive tagging system with clean interface.
