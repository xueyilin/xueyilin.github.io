Easi Game App
A HTML5 Game Platform

Role: Product Designer | Product Manager | Product Owner

Team Size: 5      Date: 2014/12-2015/01

Category: Information Architecture Product DesignStartup Project

Intro

Easi Game App is a side project for 360 PC Security's Weekly Game section which has over 600k active users and convert 100k to 360 Game Center, one of the company's most profitable product.

As more users migrate to wireless devices, the significance of PC platform is decaying. It is hard to maintain the current active users, and our team need to find new opportunity to create more value.

Goal

Create a HTML 5 game platform that transform the current 600k users on PC to mobile app.

My Role

For this project, I was the sole product designer on the team and was responsible for designing the app and the content management system from scratch. Since this project is treated as an internal startup project, I also served the role of product owner that thrives to pull resources to support the development and push the promotion of this platform.

I worked with an UI designer, an android developer, a game developer, and a database developer to make the app and games in it go live.

Research

Case Study: HTML5 Games going viral

In 2014, one HTML5 game called "Trap the Crazy Cat" has gone viral in China. By leveraging one of China's biggest social network, Wechat Moment, it has accrued 200,000,000 pageviews in merely 3 days.

By conducting case study on this game, we noticed that although the game is very mundane, there seems to be a formula that works:

  • This type of games are easy to share and play since it requires no installation

  • When a round of game ends, it assign scores that advocates social comparison

  • The game can be finished in a few minutes, thus not costing lots of productivity

  • There's no specific technique, which encourages people to try again and again

We hope that by learning from this there could be a way to make our product and go viral too.

Need Assessment: Opportunity in current market

By assessing current market environment, we also noticed that existing HTML5 games present lots of territory unexplored, despite its advantage in being able to distribute through multiple channels.

All the major competitors are taking similar approach in feature and design, with no apparent differentiation and share some common issues:

  • Stuffing all existing games into the app.

  • No champion board or any user focused features.

  • Games themselves are low quality

Data Analysis: Which of our current assets can be useful

For the PC version of weekly games, our team worked with the game development team to curate one well made flash game every week. From the data gathered for the PC version, I found that the daily ranking system and the starting screen of the game have significant effects on engagement.

Iteration

Initial Design: Focus on high-quality content.

When we first kick start the project, we can only get license for games we self-developed, so we worked with our company's game team to develop several games for the initial launch.

Based on the data of pc version, we decided to transform some of the most well received flash games to html5 games.

I started designing the app through exploring the user flow, from where it start to ending game, getting on chart and sharing with others

In the first version, we also investigated on various ways to present the game list.

We tried to utilize the high quality game by presenting the game to users right upon entrance.The rest of the games were present in a picture flow form to encourage engagement. Champion chart and sharing feature are placed at the top navigation bar to encourage user usage.

Test Run: Some design isn't fitting user context

This project was conducted under a very agile process and the first version was immediately launched after development to gather user feedback data. Through looking at the data collected we realized that:

  • The champion chart is rarely looked at.

  • Share function was almost never used after gaming.

  • Multiple feedbacks complaining there's too few games to play with.

Iteration: Redesigning the whole structure

After finding out that features like sharing and champion chart that we want to utilize is not properly emphasized. We started reconsidering the user flow and realized that these two features being placed at the top left corner makes it detached from the whole gaming process. So we decided to redesign the user flow to better fit user context.

Key Changes

Key Change 1: Picture Flow to List Layout

While doing the first version, we utilized a pinterest like layout to highlight the quality of the few games that we have available. However, as our licensed game grows, we decided to switch to version 2 to allow users to explore the amount of games we have.

Key Change 2: Featured Games to Game List

Through the test run, we found out that even though the weekly game was well received, it still didn’t escape the short life cycle nature. On top of that, the loading time of the featured game also result in a high bounce rate right upon entrance. Thus, we decided to get rid of the three tabs and the featured games, but just focus on games available

Key Change 3: Focus on Game Over Screen

Through the test run and user feedback, we realized that placing sharing and champion chart on the navbar disassociated them with the game results.So we decided to place them on the game end screen to emphasize its relation to scores

Final Design

After thorough communication with the UI designers, we also managed to get a more modern interface.

Impact

The shift from test run version to version 2 has resulted in significant increase in new account and people checking out the champion chart and replaying the game. Version 2 received 30% more daily downloads and maintained over 10% daily active rate
